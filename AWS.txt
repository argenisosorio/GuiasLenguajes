============
¿Que es AWS?
============

Amazon Web Services (AWS) es la nube más adoptada y completa en el mundo, que
ofrece más de 200 servicios integrales de centros de datos a nivel global.
Millones de clientes, incluso las empresas emergentes que crecen más rápido,
las compañías más grandes y los organismos gubernamentales líderes, están usando
AWS para reducir los costos, aumentar su agilidad e innovar de forma más rápida.

Fuente
======

https://aws.amazon.com/es/what-is-aws/?nc1=f_cc

=========================================
Amazon Simple Storage Service (Amazon S3)
=========================================

Es un servicio de almacenamiento de objetos que ofrece escalabilidad,
disponibilidad de datos, seguridad y rendimiento líderes en el sector. Clientes
de todos los tamaños y sectores pueden almacenar y proteger cualquier cantidad
de datos para prácticamente cualquier caso de uso, como los lagos de datos, las
aplicaciones nativas en la nube y las aplicaciones móviles. Gracias a las
clases de almacenamiento rentables y a las características de administración
fáciles de usar, es posible optimizar los costos, organizar los datos y
configurar controles de acceso detallados para cumplir con requisitos
empresariales, organizacionales y de conformidad específicos.

Fuente
======

https://aws.amazon.com/es/s3/

=============
Bucket en AWS
=============

En Amazon Web Services (AWS), un Bucket es un contenedor de almacenamiento en el
servicio Amazon S3 (Simple Storage Service). Los Buckets se utilizan para
almacenar y organizar datos en la nube, como archivos, imágenes, videos,
documentos, copias de seguridad y otros tipos de datos.

Características de un Bucket en AWS:

1. Contenedor de Objetos: Un Bucket puede contener un número prácticamente
ilimitado de objetos, que son los datos que almacenas. Cada objeto se compone de
datos, metadatos y una clave (un identificador único dentro del Bucket).

2. Nombre Único Global: Cada Bucket debe tener un nombre único a nivel global en
AWS, lo que significa que no puede haber dos Buckets con el mismo nombre en todo
el sistema de AWS.

3. Configuraciones Personalizables: Puedes configurar la seguridad, permisos de
acceso, políticas de versionado, políticas de ciclo de vida y configuraciones de
almacenamiento (como el tipo de almacenamiento) para cada Bucket.

4. Regiones: Al crear un Bucket, debes seleccionar una región de AWS. Esto
determina dónde se almacenarán físicamente los datos para optimizar el
rendimiento y reducir la latencia.

5. Control de Acceso: Puedes controlar quién tiene acceso a tu Bucket y los
datos almacenados en él utilizando políticas de Bucket, listas de control de
acceso (ACL), o políticas de identidad y administración (IAM).

Ejemplos de Uso:

1. Almacenar y Servir Archivos Estáticos: Como imágenes, videos, o archivos CSS
y JavaScript para un sitio web.

2. Copia de Seguridad y Archivos de Respaldo: Mantener copias de seguridad de
datos críticos.

3. Hosting de Sitios Web Estáticos: Usar S3 para hospedar sitios web que no
requieren procesamiento del lado del servidor.

4. Big Data y Procesamiento de Datos: Almacenar grandes volúmenes de datos que
pueden ser procesados posteriormente con otros servicios de AWS.

En resumen, un Bucket en AWS es un espacio de almacenamiento seguro y escalable
que puedes usar para guardar y gestionar grandes cantidades de datos de manera
sencilla.
